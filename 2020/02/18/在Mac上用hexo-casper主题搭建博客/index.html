在Mac上用hexo casper主题搭建博客

在Mac上用hexo casper主题搭建博客

前言

本文介绍将如何使用hexo和GitHub的个人主页功能搭建博客，系统为Mac系统，hexo主题是casper

题外话

说来有趣，从没想过自己会再搞一个博客出来，以为CSDN就够用了，没想到被撺掇起来，15号凌晨弄完了VPS以后抱着极大的兴趣开始搞hexo的事，有这么一段记忆其实挺好玩的。

hexo搭建的博客更像是一片伊甸园，是只为少数人所知的净土，是一面心灵的后窗。

后窗源自希区柯克于1954年拍摄的电影，片中格蕾丝凯利饰演女主角，为伊人之美所惊艳，正如片中男主角由于偷窥欲而在无意中发现的惊天秘密一样，在信息爆炸的时代秘密是无法长期保存的，无论一个人的角色大小，只要有一个合适的搜索引擎配以恰当的关键词，我们可以找到一个人从A到Z的每一面，但是也有很多历史在互联网上被隐没，变成一片人人可达而无人达的乐土，任岁月流逝，仍是一片静好。我正希望后窗有如此意义，保存一点我不为人知的秘密，向我喜欢的人敞开心门，做一片心灵的净土。

环境配置

环境上没什么好说的，网上随便一个教程可能都讲的比我恰当详细，但还是不厌其烦的赘述一下，毕竟我在环境配置上可能是浪费最多时间的。

git和ssh-key

首先安装一下git，直接使用homebrew即可安装

brew install git

然后生成ssh-key，先看看本地有没有ssh-key

ls -a ~/.ssh/

如果有的话直接

cat ~/.ssh/id_rsa.pub

没有的话

ssh-keygen -t rsa -C "your_email@example.com"

我也忘了我的ssh-key咋弄的，反正刚查网上这么写的，我也就这么信了

yourexamlpe是注册GitHub的那个邮箱，别弄错

然后生成并添加ssh-key

ssh-add ~/.ssh/id_rsa

再重复cat的命令就能看到自己的ssh-key了

接下来把ssh-key添加到GitHub上，打开个人设置

如上，完成了ssh-key部分的操作

安装nodejs，坑惨我的大头

nodejs真不是一好东西，我下了半天下下来zshrc时而能找到nvm时而找不到，第一开始我是直接用brew安装的，brew安装有两个问题，一来是brew实在太慢了，天知道我的brew多久没用过，update一下直接15分钟，不得已替换了brew的update链接，但是依然很慢，而来brew安装的要去.zshrc里改路径配置，直接用curl就没这么多傻事儿，但最坑的是我那天curl刚巧崩了，所有的GitHub链都443 connection refused了，就老老实实brew装半天装了一个残废的nvm出来，但是当时好歹也装上了，装上以后zshrc该改的也改了，hexo也装上了，那会儿已经三点了，我心满意足合上电脑上床睡觉，早上八点起来hexo一下，zsh command not found，WTF？？？？？当场抓狂，重启，curl恢复，遂用curl全部重装，一切正常

curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.29.0/install.sh | bash

简而言之上面一行顶brew里千千万万行

我记得上面那行应该是能在zshrc里自动加路径的，但是以防万一还是cat一下好了，如果没有的话，就在底下加上这么两行

export NVM_DIR="/Users/cuichengyuan/.nvm"

[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" # This loads nvm

以防万一我刚试了下nvm，还在，嗯，很好

剩下的是安装node.js

nvm install stable

node -v看一下是否安装成功

安装hexo

安装hexo还是比较顺利的

npm install -g hexo-cli

然后创建一个博客文件夹，命令hexo init blog即可完成，不一定非得叫blog，我只是照着网上的教程图省事儿用的blog，你要是愿意叫pornhub都没人管你

hexo init 你给博客起的名字

然后进入对应的文件夹（我这里为blog）

cd blog

npm install

这个过程稍微等等，可能会报错，无视就好

hexo的主要配置文件是_config.yml

这东西很神奇，因为主题也有config.yml，真不知道是咋想的

如果install好了的话，可以跑一下试试，命令是

hexo s

然后localhost 4000看看可不可以，可以的话就没问题

hexo里面常用的命令也没多少，hexo generate，生成静态文件，好像是在public文件夹里，hexo s，运行，hexo clean，删除缓存文件，有时候跑不起来clean一下试试，反正我没遇见过clean能解决的，姑且认为有用就行,详细的命令看这个人的，这人总结的很全，但有时候版本不太对，了解一下就行，hexo命令详解。

然后做点有趣的事儿，改改主题写写文章好了。

搭建博客

casper主题安装与配置

废话王终于把前面的说完了，接下来说说这个主题吧

这个主题是ghost衍生的一个主题，ghost是一个收费的框架，不建议纯玩的人使用，但是ghost主题很简洁，而且好看，个人觉得next太素了，而且有点呆板，casper的中文介绍页进去就让我眼前一亮，这里也有个小插曲，不过也是后话了

先把主题搞下来，我觉得比写文章重要，因为这个主题对post_md之类的模版是有一些修改的，就算现在直接搞也不一定能出很好的效果

这里是casper的中文主题介绍页面

首先下载这个主题，在blog文件夹下载casper主题

git clone https://github.com/xzhih/hexo-theme-casper.git themes/hexo-casper

然后把blog的config.yml的theme修改一下

theme字段应该是hexo的默认主题landscape，改成hexo-casper就好，可以进入./blog/themes的文件夹看看自己都有哪些主题

cd themes/casper

git pull

下下来最新的casper主题，不，是作者移植的casper主题

当然了，casper的官方文档在这儿呢

然后进入blog/scaffolds/文件夹，里面有一个post.md文件，casper主题需要对这个文件进行更改

cover_img是主页里的封面图，feature_img是博文post里的封面图，description和keywords我暂时还没用到，就不瞎扯了

footer_text可能有点问题，作者的博客有，但是我这里会报错，暂且搁置

casper支持的widgets比较少，下下来的只有三个，不过也足够了，一个是categories一个是tags cloud一个是recent posts

第一个可以做about page

hexo new page about

这么着就ok了，自己在source里面填就行了，然后可以看看valine

valine评论系统和访客人数

casper是支持valine的，默认的theme config里valine是true，但是valine的访客部分还是有一定修改的

valine的官方文档在这里

valine的配置看官方文档就可以了，但是对于casper主题，里面有几个小问题，头像我直接隐藏掉了，花花绿绿的不够好看，但是设置里面有一点很奇怪，正常来说评论的配置（nick mail）是meta（guest_info是老版），但是如果在设置里面直接meta没有反应，这让我很头疼，所以我直接在html里改的，html的话搜valineScript就直接可以改

这玩意儿我觉得看看应该就会，在此不再详述，同样的访客人数也是一个问题

访客人数在valine的官方文档里应该可以通过ejs直接添加代码自动加到每一篇post里，但我还真不知道怎么打开ejs这个东西，而且casper里不光是ejs，也有swig等等，所以也就懒得直接改了

文章阅读量统计valine官方文档

根据官方文档的提示，我在每一个html里找加入的位置，其实加哪里完全看个人喜好，我觉得加在日期后面比较好看，所以就找日期部分，搜索post-full-meta-date就能找到日期了

然后在底下按照官方文档的提示增加阅读量的代码，但是lean-cloud visitor count这个class有一个问题，前面会平白无故多一个·出来，这样真的不好看，而且换行上也稍微有点错乱，所以我在分类、日期后面新增加了一个data-divider，把阅读量挪出来，leancloud visitor counts的工作原理是先找id 为leancloud-visitors的东西，然后把对应的url路径输进去，取出它的信息，再填充到leancloud-visitors-count，所以这样弄下来分隔符会到阅读量的后面，也没有了多换行的问题

如此就能成功的取出阅读量并显示出来，但其实能不能显示还是靠运气的一件事儿，毕竟leancloud有时候不稳定，如果长时间出不来有两种可能，一种是弄错了，一种是leancloud卡了，检查一下网页元素，在资源里面会有leancloud visitor 这个东西，如果显示资源加载错误，可以刷新试试

这样虽然加载出了阅读量统计，但是每次在deploy以前都需要改index的代码，有点繁琐，但是ejs又懒得开，现阶段先这样将就将就

题外话

终于说到题外话了，说实话选这个主题是因为它确实挺好看的，但是它也有一个问题，由于Mac现在有黑暗模式，在黑暗模式和白日模式之间，主题会换，背景颜色啊图片通透度啊字体颜色全都会变，但是由于我不是一个喜欢光明的人（认真），我觉得黑色，深灰色或者蓝色能让我感到心情平静舒畅（蓝色纯粹是个人喜欢，蓝色也得是深蓝色或者暮色蓝），而且让网页简单大气，所以我决定删了那个白天模式尽情的拥抱黑暗

深色模式是靠查询浏览器实现的，prefers-color-scheme会返回现在的主题配色，然后对应加载不同的网页配置

所以根据这个提示，我决定找找深色模式究竟在哪，第一开始我锁定了darkmode这个css文件，但是里面修改文件没变化依然会变，但是很重要的提示信息是root，root这个东西可以控制全局的模式，所以现在就找root在哪里就好，打开css的第一个allinonecss.min.css文件，看了一看就恶心了，这玩意儿连换行都不换，让我咋看，一猜就是里面有作者的小秘密，但其他的几个文件打开以后也都不像，陷入困境

打开Safari看了看每个CSS文件，我顺手往下一拉，竟然在allinonecss.min.css2800行左右看到了root的东西，就在整个文件最下面，毕竟Safari加载这些文件的时候都会编译，整个文件就变得清晰明了起来，可算是找着了，删了root的第一部分，世界变得干净清爽

还有因为在某些纯文字文章里，背景颜色和文章背景混为一体，不甚美观，所以我决定换一下文章的背景色，但是这玩意儿都在darkmode里面，我又不能实时预览这些颜色，况且你让一个色弱去辨色就跟让一个近视1000度的人穿针一样，所以我用了数码测色计

没错就是这个可爱的Mac原生小🔧，然后呢，我用这个网站挨个像素点测颜色，最后知道了darkmode里这些东西都是对应的哪些颜色，然后改了bg-color发现并不那么好看，故作罢
